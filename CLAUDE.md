## WHAT IS TRIXIE OS

Trixie OS is the AI-orchestrated operating system that runs the entire Keep Group
business — from the first piece of content a prospect sees, to the final handover
of a completed modular home. Every tool we build is one module in this system.
They must all connect. Build everything API-first. Build everything to be extended.

This CRM (or whatever module this is) is NOT a standalone app.
It is one node in a larger connected system.

## THE FORMA PRODUCT SYSTEM

FORMA is Keep's pre-engineered productised building system. 80%+ of revenue target.

### Product tiers:

**Micro Forma — under 20m²**
Studios, home offices, wellness pods, pool houses. Often CDC-exempt.
Products: Madison, Monaco, Monaco Side, Raven, Shell, Renae (tiny home on wheels),
Wellness Pod. Price range: ~$41k–$146k.

**Duo Forma — 20m² to 70m²**
Secondary dwellings, granny flats, Airbnb, NDIS. CDC pathway. 2-module builds.
Products: Djua (1-bed $182k), Raglan (2-bed $186k), Arramont (3-bed $193k),
Karri (2-bed $172k), Yarra Hut (1-bed $167k), Madison 30 ($133k).
⭐ PRIMARY SALES FOCUS — 90-day goal is 10 Duo-Forma sales.

**Life Forma — 70m²+**
Primary homes, duplexes, low-to-mid rise. DA or CDC. 4+ modules.
Products: Acacia (4-bed, 142m², $449k). Highest margin, most complex.

**Bespoke — outside FORMA grid**
Custom design, higher cost, longer timeline. Trixie must FLAG and RECLASSIFY
any project deviating from FORMA standard sizes/finishes as bespoke.

## TRIXIE OS — 9-LAYER ARCHITECTURE

### Layer 0: Content Creation Hub — The Demand Engine
The START of everything. Drives organic demand. Feeds the pipeline.
Loop: Content → Brand → Enquiries → Projects → More content.
- 14,000+ organic Instagram followers already validated this model
- Platforms: Instagram (primary), LinkedIn, Website/blog, Email/EDM, YouTube
- Tools: Social content AI, content calendar, case study engine, EDM engine,
  L1 estimator widget (lead magnet), brand asset manager, analytics dashboard
- Agent: Content agent — monitors pipeline milestones, drafts posts,
  generates case studies, manages calendar, triggers EDM sequences

### Layer 1: Intelligence Core — The Trixie Brain
Central AI orchestration. Routes tasks to agents. Enforces business rules.
Maintains memory. Learns from outcomes. Never makes irreversible decisions
without human sign-off.
- Tools: Orchestration engine, memory store, FORMA rules engine,
  business logic layer, learning loop, human escalation manager

### Layer 2: Client Pipeline — Enquiry to Contract
Full client journey from first contact to signed contract.
CRM, lead management, estimating, proposals, contract execution.

**THE 3-TIER ESTIMATING LADDER:**
- L1 — Self-serve instant estimate (±25%) — client-facing, website widget
- L2 — Feasibility estimate (±10-15%) — generated after site visit + FORMA config
- L3 — Fixed-price contract sum (±3%) — after design resolved, goes in contract

**Pipeline stages:** enquiry → qualified → feasibility → proposal → build → delivered

- Agent: Scout — qualifies leads, scores prospects, triggers L1 estimator,
  books factory visits, triggers nurture sequences

### Layer 3: Design & Approvals
FORMA grid enforcement. CDC/DA management. Consultant coordination.
- Trixie's grid enforcer runs here — any deviation = bespoke flag
- Tools: FORMA library, grid enforcer, drawing automation, compliance checker,
  consultant coordinator, approval tracker, pre-flight QA
- Agent: Design agent — enforces FORMA grid, coordinates consultants,
  tracks approvals, triggers L3 estimate on design resolution

### Layer 4: Estimating & Procurement — The Cost Engine
Single calculator engine powering all 3 estimating tiers.
Automated supplier ordering when contract signed + deposit received.
- BOM auto-generated from FORMA config
- Supplier split by category → API/EDI/portal/email orders dispatched automatically
- Tools: Calculator core, rate card manager, FORMA pricing matrix,
  takeoff engine, supplier integration layer, PO engine, delivery scheduler,
  cost tracker, procurement scheduler

### Layer 5: Factory & Manufacturing — The Production OS
Digital OS for the factory floor. Module scheduling through QA to transport-ready.
- Tools: Production scheduler, BOM engine, QA system, module passport,
  factory dashboard, materials tracker, transport readiness tracker
- Agent: Factory agent — schedules production, manages QA, generates module
  passport, confirms transport readiness

### Layer 6: Site Delivery — Logistics & Install
Everything on site from pre-delivery coordination to handover.
Site works run CONCURRENTLY with factory manufacture (competitive advantage).
- Tools: Logistics planner, site works tracker, install tracker,
  defects manager, handover engine
- Agent: Site agent — coordinates logistics, tracks install progress,
  manages defects, generates handover docs

### Layer 7: Finance & Compliance — The Business Backbone
Financial and regulatory engine. Progress claims, cashflow, contracts,
compliance, investor reporting.
- Tools: Accounting (Xero/MYOB), progress claims engine, cashflow forecaster,
  contract engine (HIA/MBA), compliance hub, investor reporting
- Agent: Finance agent — triggers progress claims, forecasts cashflow,
  flags cost risk, produces investor reporting

### Layer 8: Content Creation Hub — Operational Detail
Operational engine producing all content across all platforms.
Connected to every layer so Trixie surfaces content from real project data.
Content types: project milestones, case studies, factory BTS, product showcases,
industry commentary, lead nurture EDM, client testimonials.

8. **Trixie Core** — master orchestrator (all layers)

**Escalation rules — Trixie MUST escalate to human for:**
- Any proposal or contract before it goes to a client
- Any project reclassified from FORMA to Bespoke
- Any cost variance >10% vs previous estimate
- Any compliance issue or regulatory flag
- Any client complaint or dispute
- Any supplier failure affecting production schedule
- Any financial commitment above defined thresholds

## ARCHITECTURE PRINCIPLES
1. Single source of truth — all data in Trixie, no floating spreadsheets
2. Event-driven — every action triggers events other agents can respond to
3. Build custom only where necessary — integrate existing tools where possible
4. API-first — every tool must expose an API
5. Human-in-the-loop at stage gates — autonomous within stages, sign-off between them
6. Content feeds pipeline feeds content — Layer 0 and Layer 2 in continuous loop
